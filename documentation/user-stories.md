# 📄 User Stories (Agile Format)

## 💰 Money Tracker

---

## 📌 Overview

This document defines user stories with priority and acceptance criteria to ensure clear understanding of system functionality.

---

## 🧩 User Stories Table

| ID    | User Story                                                                               | Priority | Acceptance Criteria                                                      |
| ----- | ---------------------------------------------------------------------------------------- | -------- | ------------------------------------------------------------------------ |
| US-01 | As a user, I want to register so that I can create an account                            | High     | User can enter name, email, password and account is created successfully |
| US-02 | As a user, I want to log in so that I can access my account                              | High     | User enters valid credentials and is logged in                           |
| US-03 | As a user, I want to add an expense so that I can track my spending                      | High     | Expense is saved with amount, category, date                             |
| US-04 | As a user, I want to view my expenses so that I can monitor my spending                  | High     | System displays list of all expenses                                     |
| US-05 | As a user, I want to delete an expense so that I can remove incorrect data               | Medium   | Selected expense is removed successfully                                 |
| US-06 | As a user, I want to categorize expenses so that I can analyze spending                  | High     | Expense is linked to selected category                                   |
| US-07 | As a user, I want to see total monthly expenses so that I know my spending               | High     | System calculates and displays total amount                              |
| US-08 | As a user, I want to view category-wise breakdown so that I understand spending patterns | High     | System shows category-wise amount and percentage                         |
| US-09 | As a user, I want to set a budget so that I can control my expenses                      | Medium   | User can set monthly budget value                                        |
| US-10 | As a user, I want alerts when budget exceeds so that I avoid overspending                | Medium   | System shows warning when limit is crossed                               |
| US-11 | As a user, I want insights so that I can improve financial habits                        | Low      | System displays spending insights                                        |
| US-12 | As a user, I want to filter expenses so that I can find specific data                    | Medium   | User can filter by date and category                                     |

---

## 📌 Priority Levels

* **High** → Core features (must have)
* **Medium** → Important but not critical
* **Low** → Optional / enhancement features

---

## 📌 Conclusion

This structured format helps in prioritizing development tasks and ensures clarity in implementation and testing of the Money Tracker system.
